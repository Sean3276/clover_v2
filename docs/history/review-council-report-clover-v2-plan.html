Traceability

Panel findings → D-decisions (Rev B)

  • D1 Basic-auth dying → OAuth2/XOAUTH2 default
  • D2 Threads span folders / UID unstable → cross-folder + Message-ID key
  • D3 Phase-2 undefined → invocation/model/batching + decisions schema
  • D4 Creds + LLM egress → keyring, redaction, consent, local option
  • D5 Link-share over-promised → carry-over + chase
  • D6 No accuracy guarantee → council + QAQC + golden-set eval
  • D7 Prototype too big → capture core + thin UI

Field themes (freq) → S-decisions (Rev C)

  • S1 LLM egress confidentiality (3/10, loudest) → tiered egress + consent log
  • S2 No accuracy/cost numbers (2/10) → published ≥90%/85% + cost in UI
  • S3 Retention/compliance thin (1/10, high-weight) → immutable .eml + legal-hold
  • S4 SharePoint deferred (1/10) → optional cookie-reuse adapter
  • S5 Scale/brand asks → kept single-operator scope
Demands the Lead rejected: Marketing's brand/shareable-output features (out of scope for an internal tool); HR's multi-user/SSO now (premature for a single-operator prototype); full SharePoint crawl as a Phase-1 blocker (would stall the first testable ship). Each would bloat or delay without serving the core operator.
