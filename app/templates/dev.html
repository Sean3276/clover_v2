Add a model

Set up the Claude CLI — uses your Claude subscription, no API key.

  1. Install: npm i -g @anthropic-ai/claude-code
  2. Sign in once: run claude in a terminal and log in.
  3. Check it works: claude -p "hi"
  4. Set Model below to sonnet, opus, or haiku, then Save model and Activate it.

Set up the Codex CLI (GPT-5.x) — uses your ChatGPT account, no API key.

  1. Install the Codex CLI and sign in with ChatGPT (follow Codex's own install docs).
  2. Check it works: codex exec -m gpt-5.5 "hi"
  3. Set Model below to gpt-5.5 (or another Codex model), then Save model and Activate it.

Set up Ollama (local) — FREE, private (email never leaves your machine), no daily/rate cap.

  1. Install from ollama.com, then start it: ollama serve.
  2. Pull a model — for mixed English/Chinese on a big-RAM box an MoE is ideal: ollama pull qwen3.6:35b-a3b (or the higher-quality qwen3.6:35b-a3b-q8_0 if you have the RAM).
  3. Check it works: ollama run qwen3.6:35b-a3b "hi"
  4. Set Model below to the exact tag (e.g. qwen3.6:35b-a3b), then Save model and Activate. Comprehension is slow but unlimited — it grinds in the background.

Stub — a deterministic offline backend for testing. No setup and no AI; it returns placeholder output.

Comprehension concurrency

How many threads Phase 3 comprehends in parallel. A higher number clears the backlog faster, but throughput is ultimately capped by the backend — a local model serializes on one GPU, and CLI backends are rate-limited. Start at 2–4 and watch the error count.

AI timeout

How long to wait for a single AI step before giving up. The decomposed pipeline makes several cold-start CLI calls per thread, so a slow model can need more than the 300-second default. Raise this if comprehension times out; lower it to fail faster on a stuck backend.
